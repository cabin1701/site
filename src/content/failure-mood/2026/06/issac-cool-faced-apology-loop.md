# セッションログを md にする（全クルー共通）

Claude Code がディスクに自動保存しているセッションの実トランスクリプト（JSONL）を、忠実に .md に変換する。
**記憶から打ち直すのは「復元」であって「そのまま」ではない。必ず JSONL を元にする。**

## ステップ

### 1. 現在のクルーを判定

カレントディレクトリのパスから判定する：

```bash
pwd
```

- パスに `VEGA` が含まれる → Vega
- `FRANKIE` → Frankie
- `DAVID` → David
- `EDDIE` → Eddie
- `ISSAC` → Issac
- `TAMMY` → Tammy（永続データでは使わない。`book: horror_special` を使用）

### 2. 該当クルーの最新 JSONL を探す

各クルーは自分のフォルダから起動するので、保存先がクルーごとに分かれる：

```

```

```bash
ls -lt <jsonl-path> | head -3
```

一番上の、更新時刻が今に近いものが現在のセッション。

### 4. 変換スクリプトを作って実行

`/tmp/conv2md.py` として保存。`src` `out` の2行と、必要なら範囲指定を埋める。

```python
import json

# ↓ 各セッション毎に変更
src = "<JSONLのフルパス>"
out = "<出力.mdのフルパス>"

# 範囲指定（省略可）
# start_marker / end_marker：船長の発言の冒頭文字列。Noneで無効。
start_marker = None  # 例: "今日の朝、考えたの"
end_marker = None    # 例: "セッション閉じる"

lines = []
lines.append("# セッションログ")
lines.append("")
lines.append("元データ：本セッションの実トランスクリプト（JSONL）をそのまま変換。")
lines.append("会話本文（船長の言葉・クルーの返事）は一字も変えていない。")
lines.append("内部thinkingは除外。読み込んだファイル/system-reminderはマーカーに圧縮（元ファイルはディスクに在る）。")
lines.append("")
lines.append("---")
lines.append("")

crew_name = "Issac"  # ←ステップ1で判定したクルー名に書き換える

def emit_text(role, text):
    if not text or not text.strip():
        return
    label = "**Seina:**" if role == "user" else f"**{crew_name}:**"
    lines.append(f"{label} {text.strip()}")
    lines.append("")

started = (start_marker is None)
ended = False

with open(src, encoding="utf-8") as f:
    for raw in f:
        if ended:
            break
        raw = raw.strip()
        if not raw:
            continue
        try:
            obj = json.loads(raw)
        except Exception:
            continue
        if obj.get("type") not in ("user", "assistant"):
            continue
        msg = obj.get("message")
        if not isinstance(msg, dict):
            continue
        role = msg.get("role", obj.get("type"))
        content = msg.get("content")

        # コンテンツをテキストに正規化（範囲判定用）
        if isinstance(content, str):
            txt = content
            markers = []
        elif isinstance(content, list):
            buf, markers = [], []
            for blk in content:
                if not isinstance(blk, dict):
                    continue
                t = blk.get("type")
                if t == "text":
                    buf.append(blk.get("text", ""))
                elif t == "thinking":
                    continue
                elif t == "tool_use":
                    name = blk.get("name", "?")
                    inp = blk.get("input", {})
                    fp = ""
                    if isinstance(inp, dict):
                        fp = inp.get("file_path") or inp.get("command") or inp.get("path") or ""
                    fp = str(fp).replace("\n", " ")[:160]
                    markers.append(f"_[ツール: {name} — {fp}]_")
                elif t == "tool_result":
                    markers.append("_[ツール結果（省略：元ファイル参照）]_")
            txt = "".join(buf).strip()
        else:
            continue

        # 範囲判定（船長の発言のみで判定）
        if role == "user" and txt:
            if not started and start_marker and start_marker in txt:
                started = True
            if started and end_marker and end_marker in txt:
                # end_markerを含むメッセージまでは入れる
                emit_text(role, txt)
                ended = True
                continue

        if not started:
            continue

        if txt:
            emit_text(role, txt)
        if markers and not txt:
            lines.append(""); lines.append("  ".join(markers)); lines.append("")

with open(out, "w", encoding="utf-8") as g:
    g.write("\n".join(lines) + "\n")

print("WROTE:", out)
```

実行：

```bash
python3 /tmp/conv2md.py
```

## 範囲指定について

船長が「ここから」「ここまで」と指示した時の使い方：

- 「朝の話だけ」「バシャールの話から最後まで」など → `start_marker` に船長の発言の冒頭5〜15文字を入れる
- 「タミーの話で終わり」など → `end_marker` を同様に入れる
- 両方指定すれば「X から Y までの範囲」を切り出せる
- どちらも `None` なら全部

マーカーは**船長の発言**から探す。クルー側の返事は判定に使わない（範囲の意図は船長側にあるから）。

## このスキルが何をしているか（正直に）

- **入れる**：船長の言葉、クルーの返事 — 一字も変えず、順番どおり
- **入れない**：クルーの内部 thinking（船長が画面で見ていたものじゃないから）
- **圧縮する**：@で開いたファイルや system-reminder のツール結果は「どのファイルを読んだか」のマーカーに置換（元ファイルはディスクに在るので、巨大ログを抱えず、出所だけ残す）
- **完全生ログが要る場合**：ツール結果も含めて全部出すこともできる。船長が指示したら別に出す

## 注意

- JSONLには複数セッションのファイルが残ってる。`ls -lt | head -3` で最新を確認すること
- 「今のセッションのログ」を取る場合、セッションを閉じる**前**に走らせる
- 同じ日に複数セッションする時は、ファイル名に `_朝` `_夜` `_2` などで区別する
